## 1. 環境構築

### 1.1 必要要件

| 項目 | 要件 |
|------|------|
| Python | 3.10以上 |
| OS | macOS / Linux |
| ネットワーク | インターネット常時接続 |
| メモリ | 1GB以上推奨 |

### 1.2 APIキーの取得

| サービス | 取得先 | 無料枠 |
|----------|--------|--------|
| OANDA | [OANDA fxTrade](https://www.oanda.com/) Practice口座開設 | デモ口座無料 |
| Google Gemini | [AI Studio](https://aistudio.google.com/) | ~20 req/日 |
| Groq | [Groq Console](https://console.groq.com/) | 100,000 TPD |
| LINE Messaging | [LINE Developers](https://developers.line.biz/) | 無料（200通/月） |
| ImgBB | [ImgBB API](https://api.imgbb.com/) | 無料 |

### 1.3 インストール手順

```bash
# 1. リポジトリをクローン
git clone <repository-url>
cd AiTradeTool

# 2. 依存パッケージのインストール
pip install -r requirements.txt

# 3. 環境変数ファイルの作成
cp .env.example .env

# 4. .env を編集してAPIキーを設定
```

### 1.4 .env 設定

```bash
# 必須設定
DATA_SOURCE=oanda                    # oanda または twelvedata
OANDA_API_TOKEN=your_oanda_token     # OANDA APIトークン
OANDA_ACCOUNT_ID=your_account_id     # OANDA アカウントID
OANDA_ENVIRONMENT=practice           # practice（デモ） or live

# AI分析（推奨）
GEMINI_API_KEY=your_gemini_key       # Gemini API キー
GROQ_API_KEY=your_groq_key           # Groq API キー

# 通知（任意）
LINE_CHANNEL_ACCESS_TOKEN=your_token # LINE通知用
LINE_USER_ID=your_user_id            # LINE送信先ユーザーID
IMGBB_API_KEY=your_imgbb_key         # チャート画像アップロード用

# メール通知（任意）
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASSWORD=your_app_password
EMAIL_FROM=your_email@gmail.com
EMAIL_TO=your_email@gmail.com
```

## 2. 起動・停止

### 2.1 ペーパートレードボット

```bash
# フォアグラウンド起動
python main.py

# バックグラウンド起動
nohup python main.py > /dev/null 2>&1 &

# PID確認
ps aux | grep "python main.py"

# 停止
kill <PID>
# または Ctrl+C（フォアグラウンド時）
```

### 2.2 Webダッシュボード

```bash
# デフォルトポート（5000）
python src/dashboard.py

# ポート指定
python src/dashboard.py --port 8080

# バックグラウンド起動
nohup python src/dashboard.py --port 8080 > /dev/null 2>&1 &

# アクセス
open http://localhost:8080
```

### 2.3 ボット + ダッシュボード同時起動

```bash
# ボットをバックグラウンドで起動
nohup python main.py > /dev/null 2>&1 &

# ダッシュボードをバックグラウンドで起動
nohup python src/dashboard.py --port 8080 > /dev/null 2>&1 &
```

### 2.4 macOS 常駐化（LaunchAgent）

```bash
chmod +x deploy/install_mac.sh
./deploy/install_mac.sh
nano ~/.ai-fx-bot/.env                                        # APIキー設定

# 起動
launchctl load ~/Library/LaunchAgents/com.ai-fx-bot.plist

# 停止
launchctl unload ~/Library/LaunchAgents/com.ai-fx-bot.plist

# ログ監視
tail -f ~/.ai-fx-bot/logs/trading.log
```

### 2.5 Linux 常駐化（systemd）

```bash
sudo chmod +x deploy/install.sh
sudo ./deploy/install.sh
sudo nano /opt/ai-fx-bot/.env                                 # APIキー設定

# 起動 / 停止 / 状態確認
sudo systemctl start ai-fx-bot
sudo systemctl stop ai-fx-bot
sudo systemctl status ai-fx-bot

# ログ監視
journalctl -u ai-fx-bot -f
```

## 4. 日常運用

### 4.1 毎日の確認事項

| 確認項目 | 方法 | 目安 |
|---------|------|------|
| ボット稼働確認 | `ps aux \| grep main.py` | プロセス存在 |
| ダッシュボード | ブラウザで http://localhost:8080 | 表示正常 |
| オープンポジション | ダッシュボード or `--status` | ペア別確認 |
| 含み損益 | ダッシュボード UNREALIZED P&L | 許容範囲内 |
| ドローダウン | ダッシュボード DRAWDOWN | < 10% |
| Gemini残量 | ダッシュボード Gemini残量表示 | 使い切り前に確認 |
| ログ異常 | `tail -100 logs/trading.log` | エラーなし |

### 4.2 Gemini API予算管理

| 項目 | 値 |
|------|-----|
| 日次上限 | 18回（予備2回） |
| 呼び出しタイミング | 4H足確定時（UTC 0,4,8,12,16,20時） |
| 呼び出し条件 | アルゴがパターン検出時のみ |
| リセット | 毎日自動リセット |

**予算超過時の動作**: Geminiスキップ → キャッシュされた前回分析結果を使用。
アルゴリズム + Groqの2票で判断を継続。

### 4.3 Groq API使用量

| 項目 | 最適化前 | 最適化後 |
|------|---------|---------|
| 日次呼び出し | ~2,016回 | ~50回 |
| 方向性投票 | 毎5分 | 4H足確定時のみ（キャッシュ） |
| レジーム分析 | 毎5分 | 1時間キャッシュ |

### 4.4 ログファイル

| ファイル | 内容 | ローテーション |
|---------|------|--------------|
| `logs/trading.log` | メインログ（INFO以上） | 50MB / 1週間 |
| `logs/trades.log` | トレード実行ログ | — |
| `data/ai_analysis_log.json` | AI分析結果 | — |

## 5. パラメータ調整

### 5.1 設定ファイル

全トレード設定は `config/trading_config.yaml` で管理。

### 5.2 主要パラメータと調整ガイド

#### アクティブペア変更

```yaml
# config/trading_config.yaml
trading:
  active_pairs:
    - "USD_JPY"
    - "EUR_USD"
    - "GBP_JPY"
    # - "EUR_JPY"    # コメントアウトで無効化
```

#### リスクパラメータ

```yaml
risk_management:
  risk_per_trade_pct: 1.0        # 1トレードリスク（推奨: 0.5-2.0%）
  stop_loss_atr_multiple: 2.0    # SL距離（推奨: 1.5-3.0）
  take_profit_rr_ratio: 2.0      # R:R（推奨: 1.5-3.0）
  max_open_positions: 6          # 同時ポジション上限
  max_positions_per_pair: 2      # ペア別上限
  trailing_stop_atr_multiple: 1.5 # トレーリングストップ距離
```

#### 信頼度閾値

```yaml
risk_management:
  confidence_threshold: 55       # 初期閾値（50-65の範囲で自動調整）
  confidence_threshold_min: 50   # 下限（これ以下には下がらない）
  confidence_threshold_max: 65   # 上限（これ以上には上がらない）
```

#### 適応的リスクの攻撃性

```yaml
risk_management:
  adaptive:
    loss_scale_factor: 0.75      # 連敗時の縮小率（0.5=保守的, 0.9=攻撃的）
    max_consecutive_losses: 5    # 強制停止連敗数
    performance_window: 20       # 評価トレード数
```

#### サイクル間隔

```yaml
execution:
  check_interval_seconds: 300    # 5分（推奨: 300-600）
```

## 6. トラブルシューティング

### 6.1 ボットが起動しない

| 症状 | 原因 | 対処 |
|------|------|------|
| "Failed to connect to OANDA" | APIキーまたはアカウントID不正 | `.env` の `OANDA_API_TOKEN` と `OANDA_ACCOUNT_ID` を確認 |
| ImportError | パッケージ未インストール | `pip install -r requirements.txt` |
| "GEMINI_API_KEY が未設定" | 環境変数不足 | `.env` に `GEMINI_API_KEY` を設定 |
| "GROQ_API_KEY が未設定" | 環境変数不足 | `.env` に `GROQ_API_KEY` を設定（任意） |

### 6.2 API エラー

| エラー | 原因 | 対処 |
|--------|------|------|
| Gemini 429 | レート制限超過 | 自動でキャッシュに切替。予算管理で通常は発生しない |
| Groq 429 | TPD超過 | キャッシュ使用で自動継続。通常は最適化済みで発生しない |
| OANDA 401 | トークン失効 | 新しいAPIトークンを取得して `.env` 更新 |
| OANDA 503 | サーバーメンテナンス | 自動リトライ。長時間続く場合はOANDAステータス確認 |

### 6.3 ダッシュボードの問題

| 症状 | 原因 | 対処 |
|------|------|------|
| 数値が更新されない | キャッシュ | 30秒待つか、ブラウザをリロード |
| ペアが表示されない | ボット未起動 | ボットを起動してサイクル完了を待つ |
| "0.0%" 表示 | ボットのサイクルログ未生成 | ボット起動後、最初のサイクル完了を待つ |
| ポート競合 | 別プロセスが使用中 | `--port` で別ポートを指定 |

### 6.4 ポジション関連

| 症状 | 原因 | 対処 |
|------|------|------|
| 再起動後にポジションが消えた | DB復元失敗 | `data/positions.db` の存在確認 |
| ポジションがOANDAと不一致 | ペーパーモードの状態ズレ | `--fix-sltp` でSL/TP再同期 |
| エントリーしない | エントリーゲート | 4H足確定時刻（日本時間: 9, 13, 17, 21, 1, 5時）まで待つ |

### 6.5 プロセスの強制再起動

```bash
# 全プロセス確認
ps aux | grep python | grep -E "main.py|dashboard.py"

# 停止
kill <PID>

# 再起動
nohup python main.py > /dev/null 2>&1 &
nohup python src/dashboard.py --port 8080 > /dev/null 2>&1 &
```

## 7. バックテスト運用

### 7.1 基本的なバックテスト

```bash
# 全アクティブペア（OANDA APIデータ: 約3ヶ月分）
python main.py --backtest

# 特定ペア
python main.py --backtest --instrument USD_JPY
```

### 7.2 長期バックテスト（yfinance）

```bash
# 2年分（最大）
python main.py --backtest --download --years 2

# 特定ペア + 長期
python main.py --backtest --instrument EUR_USD --download --years 2
```

### 7.3 外部CSVバックテスト

```bash
python main.py --backtest --instrument USD_JPY --csv data/my_data.csv
```

CSVフォーマット: `datetime, open, high, low, close, volume`

### 7.4 結果の読み方

バックテストは **Fixed（固定リスク）** と **Adaptive（適応的リスク）** を自動比較:

| 指標 | 意味 | 目標 |
|------|------|------|
| 勝率 | 勝ちトレードの割合 | ≥ 60% |
| PF (Profit Factor) | 総利益 ÷ 総損失 | ≥ 1.2 |
| 最大DD | 最大ドローダウン | ≤ 10% |
| シャープレシオ | リスク調整後リターン | ≥ 1.0 |
| 最大連敗 | 最長連続負け | ≤ 20 |

## 8. 通知設定

### 8.1 LINE通知の設定

1. [LINE Developers](https://developers.line.biz/) でMessaging APIチャネル作成
2. Channel Access Token（長期）を発行
3. チャネル基本設定からYour User IDを取得
4. 作成されたLINE公式アカウントを友だち追加
5. `.env` に設定:

```bash
LINE_CHANNEL_ACCESS_TOKEN=your_channel_access_token
LINE_USER_ID=your_user_id
IMGBB_API_KEY=your_imgbb_key  # チャート画像送信に必要
```

### 8.2 通知の種類

| 通知 | タイミング | 内容 |
|------|-----------|------|
| ボット起動 | 起動時 | モード、対象ペア |
| トレード実行 | エントリー時 | ペア、方向、価格、SL/TP |
| ポジションクローズ | クローズ時 | 損益、理由 |
| ドローダウン警告 | DD 3/5/7%到達 | 現在DD、リスク縮小率 |
| デイリーサマリー | 日次 | 日次損益、勝率、トレード数 |
| Geminiチャート | 分析時 | チャート画像 + AI分析結果 |

## 9. 4H足確定時刻表

エントリー判断は4H足確定時のみ。以下の時刻にアルゴリズム + Gemini + Groq の全分析が実行される。

| UTC | 日本時間 (JST) | 備考 |
|-----|---------------|------|
| 00:00 | 09:00 | 東京市場オープン |
| 04:00 | 13:00 | 東京午後 |
| 08:00 | 17:00 | ロンドン市場オープン |
| 12:00 | 21:00 | NY市場オープン |
| 16:00 | 01:00 | NY午後 |
| 20:00 | 05:00 | 早朝（薄商い） |

**1日最大エントリー判定回数**: 6回 × 6ペア = 36回
**Geminiが呼ばれるのは**: パターン検出時のみ（最大18回/日）
